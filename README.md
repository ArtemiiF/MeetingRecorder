# Meeting Recorder

Локальная запись встреч → транскрипт → сводка по промпту. Всё на машине, без облака.

## Установка (.app)

1. Скачай `Meeting Recorder-X.Y.Z-arm64.dmg` из [последнего релиза](https://github.com/ArtemiiF/MeetingRecorder/releases/latest) → перетащи в Applications.
2. **Gatekeeper заблокирует первый запуск** («Apple could not verify…») — приложение подписано ad-hoc, без нотаризации Apple. Это ожидаемо. Жми **Done** (не «Move to Trash») и разблокируй одним из способов:
   - **Терминал:** `xattr -rd com.apple.quarantine "/Applications/Meeting Recorder.app"` → открывай обычным двойным кликом.
   - **Настройки:** Системные настройки → Конфиденциальность и безопасность → внизу «"Meeting Recorder.app" was blocked» → **Open Anyway**.
3. Первый запуск: setup-стена проведёт через установку бэкенда (~1.3 ГБ) и скачивание моделей; macOS спросит разрешения на микрофон и системный звук.

**Дальше руками ничего не ставится**: Настройки → Обновления → «Проверить обновления» — приложение скачивает и устанавливает новые версии само (карантин-предупреждения при этом не появляются). После обновления macOS может заново спросить разрешения — ad-hoc-подпись меняется между сборками.

> Убрать предупреждение Gatekeeper полностью можно только платной подписью (Apple Developer Program, $99/год: Developer ID + нотаризация). Бонусом она чинит слёт разрешений между обновлениями и keychain-хранение токена в упакованном приложении.

## Запуск (dev)

```bash
cd app
npm install
npm start
```

Первый запуск берёт пресеты и пути из `presets.example.json`; личный `presets.json` (в git не попадает) пишется при первом сохранении настроек. Можно сразу скопировать и отредактировать: `cp presets.example.json presets.json`.

Нужно для полного цикла:
- **LM Studio** запущен на `localhost:1234` (для сводки).
- **HF-токен** в поле приложения — для диаризации (определение спикеров). Без него спикеры по таймкодам.

## Захват звука

- **Микрофон** — Python (pyaudio), устройство по умолчанию или выбранное в UI. Разрешение: микрофон.
- **Системный звук** (голоса остальных в Zoom/Meet) — AudioTee (Core Audio tap, macOS 14.2+), **без виртуального драйвера**.
- На стопе mic.wav + system.wav сводятся ffmpeg `amix` → mixed.wav (его обрабатывает пайплайн; обе дорожки сохраняются рядом).

### ⚠️ Разрешение на системный звук

AudioTee требует TCC-разрешение **«System Audio Recording Only»**.

- **Dev-запуск (`npm start`):** macOS может не показать промпт автоматически из незапакованного Electron. Если в статусе «Системный звук недоступен» — добавить вручную:
  Настройки → Конфиденциальность и безопасность → **Запись экрана и системного звука** → раздел **System Audio Recording Only** (не верхний!) → добавить Electron/терминал.
- **Запакованный билд:** добавить в `Info.plist` ключ `NSAudioCaptureUsageDescription` (текст запроса) и подписать приложение — бинарь AudioTee наследует подпись родителя. См. README пакета `audiotee` → раздел Code signing.

Если системный звук недоступен — запись продолжается **только с микрофона** (mixed = mic).

## Структура

- `backend.py` — Python CLI (JSON-stdout): `devices` / `record` (mic) / `mix` / `process` (transcribe→diarize→LLM).
- `main.js` — Electron main: AudioTee (system) + WavWriter, спавн mic-рекордера и пайплайна, IPC.
- `renderer/` — UI.
- `lib/mainutil.js` — чистые хелперы main.js (WAV, VU-RMS, cache-key, history, token-blob) — вынесены для тестируемости.
- Тесты: `cd app && npm test` (22 JS через `node:test`+jsdom — renderer/main-логика; 54 Python — backend/pipeline).

## Возможности

- Запись (микрофон + системный звук) и импорт готового аудио.
- Транскрипция + диаризация (опц.) + сводка по редактируемому промпту (пресеты).
- **Язык**: ru / en / auto.
- **Только транскрипт** (без LLM-сводки).
- **Resume по этапам**: транскрипция/диаризация кэшируются; повтор (↻) не пересчитывает их заново. «Заново» сбрасывает кеш.
- **Остановка** обработки (кеш переживает отмену).
- **VU-индикатор** уровня mic+system при записи.
- **История записей** — открыть/переобработать прошлые.
- Прогресс по этапам с раздельными логами и статусом (🟢/🔴/⚪).
- **Preflight-панель** готовности: LM Studio / разрешения / ffmpeg / модель / HF-токен — до записи.
- **Имена спикеров**: дефолт «Спикер 1/2…», переименование в результате (переписывает заметку). Авто-«Я» из mic-дорожки — отложено (VAD сжимает время, нужна рекомпозиция).
- **Заголовки**: тема из LLM → `title:` в frontmatter + лейбл в истории (поиск через месяцы).

## Хранение

- **Заметки `.md` в Obsidian — источник правды.** Метаданные (`title`, `template`, `language`, `date`) во frontmatter.
- **`index.db`** (SQLite, Python `sqlite3` stdlib) — **производный индекс** для вкладки «История»: `backend.py history` сверяется с папкой Obsidian (добавляет новые из frontmatter, дропает удалённые), не источник правды. Удалишь — пересоберётся. Без нативных JS-модулей (нет ребилда под Electron).

## Безопасность / приватность

- **HF-токен** хранится зашифрованным через OS Keychain (`safeStorage`) в `app/.secret`, не в `presets.json`.
- `.gitignore` исключает `presets.json`, `.secret`, `node_modules`.
- Temp-файлы записей/кеша в `os.tmpdir()/meeting-recorder` чистятся при старте (старше 7 дней).
- ⚠️ Если HF-токен когда-либо был в git/plaintext — **отозвать** на huggingface.co/settings/tokens.
